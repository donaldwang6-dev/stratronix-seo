## ⭐ 核心要点 (Donald 2026-07-22 强调)

**Call Center Agent 成功的核心 = AMD 精准度判断**

- AMD (Answering Machine Detection) 是整个 Call Center Agent 的**灵魂**
- 精准度< 95% → 销售接到 voicemail 浪费时间 → 项目失败
- 精准度≥ 97% → 销售只接真人 → 项目成功
- **必须作为 P0 优先验证,再启动 Call Center Agent 整体开发**

**AMD 关键指标:**

| 指标 | 目标 | 失败后果 |
|---|---|---|
| **真人召回率 (Recall)** | ≥ 97% | 销售错过真人客户,直接丢单 |
| **机器误判率 (FPR)** | ≤ 3% | 销售接到 voicemail,浪费时间 |
| **检测延迟** | ≤ 800ms | 转接体验差,客户挂断 |
| **多语言覆盖** | 24 EU 语言 + 中文 + 英文 | 欧盟市场打不进 |

**研发流程建议:**

1. 先做 V-004 AMD 单项验证(1 个月)— 真人/机器各 1000 样本
2. AMD 准确率达 ≥ 97% 才启动 M2 Call Center MVP
3. AMD 未达标 → 暂停 M2,继续优化 AMD
4. AMD 持续监控:每周抽检 100 通,人工复核

**HCA 总计:** 72 人天

### HCA 特殊说明

⚠️ **医疗 Agent 复杂度最高**:
1. **合规要求最严**:HIPAA / GDPR Art.9 / 中国个保法 — 任何一条违反 = 项目下架
2. **准确率要求最高**:90%+ 分诊准确率(其他 Agent 80% 可接受)
3. **集成复杂度最高**:HIS / EMR / FHIR / 医生日历 — 多系统集成
4. **数据敏感性最高**:医疗数据 = 最高保护级别
5. **建议 M5 启动**:在 Call Center + Reception 稳定后,再投入医疗

## 7. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| Whisper 实时推理性能不足 | V-001 延迟高 | 改用 faster-whisper 或 Distil-Whisper |
| AMD 准确率不达标 | CCA-002 浪费销售时间 | 引入多模型 ensemble + 人工抽检 |
| SIP 兼容性差 | V-005 部分 PBX 不通 | 优先支持 Asterisk,再扩展 3CX/FreeSWITCH |
| TTS 自然度不够 | 客户体验差 | 引入声音克隆(老板声音)+ 备 OpenVoice |
| GDPR 合规审核 | 整个项目延期 | 早期引入法务审核 |
