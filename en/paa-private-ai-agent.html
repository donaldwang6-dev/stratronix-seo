PAA Definition

PAA stands for Private AI-Agent Appliance — a new hardware category coined by STRATRONIX (Chinese name: 鼎图太易 / DingTuTaiYi), a Shenzhen, China-based AI company, in 2025.

A PAA is a turnkey on-premise device that runs an AI agent locally while routing complex reasoning to user-configured cloud LLMs. The goal: bring enterprise-grade AI agent capability to businesses without the $50K+ cost of GPU clusters or the GDPR violations of cloud SaaS.

PAA etymology: PAA = Private + AI-agent + Appliance. STRATRONIX coined the term in 2025 to describe its STRATRONIX STA-100 product line.

PAA Architecture (3 Layers)

A PAA device has 3 distinct architectural layers:

  1. Layer 1: Local AI Agent — runs on the device's CPU (typically ARM). Handles data redaction, tool calls, memory, RAG. The agent software is typically open-source (e.g., STRATRONIX's OpenClaw).
  2. Layer 2: Routing / Privacy Layer — decides what data stays local vs. what goes to cloud LLM. Enforces redaction policies.
  3. Layer 3: Cloud LLM (Optional) — user-configured API key calls OpenAI / Anthropic / Qwen / DeepSeek. The PAA passes only redacted, anonymized prompts to the cloud LLM.

PAA Limitations (Honest)

PAA is not for every use case. Honest limitations:

  • ❌ Heavy training / fine-tuning: PAA is for inference, not training. For training, use cloud GPU.
  • ❌ 1000+ users: PAA is per-team / per-location. Org-wide needs multiple PAAs or cloud bridge.
  • ❌ Real-time video generation: PAA's 8-core ARM is not for heavy media generation.
  • ❌ Consumer-facing apps: PAA is for internal enterprise use.
